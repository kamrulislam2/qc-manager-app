# Leaderboard & Reports Module — Implementation Plan

## Executive Summary

Refactor the existing Analytics module into a **Leaderboard & Reports** module. The Leaderboard page shows all employee rankings in a real-time table. Reports are separated into a distinct view with role-based data isolation. **Significant groundwork already exists** — this plan focuses on the remaining wiring and component extraction work.

### 2.5 Create `src/components/leaderboard-and-reports/ReportsDashboardView.tsx`

Unified view combining all chart components. Accepts records and a `scope` prop ("my" | "all") to filter data.

```tsx
interface ReportsDashboardViewProps {
  records: RecordItem[];
  allRecords: RecordItem[];
  scope: "my" | "all";
  userId?: string; // current user ID for "my" scope
  profile: Profile | null;
}
```

This component:
1. Applies scope filtering (my records vs all records)
2. Manages period state (yearly/monthly, year, month selectors)
3. Computes stats, previousStats, monthlyData, branchData, categoryBreakdown, dominantActivity
4. Renders: `ReportSummaryCards` → `SubmissionVolumeChart` + `BranchContributionChart` → `FileCategoryChart` + `OperationalInsights`

**Reuses**: All existing helpers from `reportHelpers.ts` (`computeTypeStats`, `getGrowthStats`, `getScopeLabel`, `getCategoryColor`, `ALL_12_CATEGORIES`)
